# FICBANK Security Control Implementation and Validation

📄 Full Report: [View PDF](PASTE-YOUR-GITHUB-LINK-HERE)

## Overview
This project focuses on applying and validating security controls within a financial institution (FICBANK). Based on previously identified vulnerabilities, security controls were implemented to reduce risk, improve system hardening, and strengthen overall security posture.

## Tools & Concepts Used
- NIST-based security control concepts
- System hardening techniques
- Access control and authentication
- Network segmentation
- Logging and monitoring

## Key Findings
- Weak access control and authentication mechanisms
- Lack of centralized logging and monitoring visibility
- Misconfigured systems increasing attack surface
- Inadequate validation of file uploads and inputs

## Controls Implemented

### Access Control & Authentication
- Enforced least privilege access across systems
- Implemented multi-factor authentication (MFA) for secure access

### Network Security
- Applied network segmentation to limit lateral movement
- Strengthened firewall configurations to control traffic

### System Hardening
- Applied patch management and system updates
- Disabled unnecessary services and configurations

### Monitoring & Logging
- Enabled centralized logging for visibility
- Implemented monitoring for suspicious activity detection

### Data Protection
- Applied file validation controls to prevent malicious uploads
- Strengthened encryption and backup strategies

## Validation
- Tested implemented controls to ensure effectiveness
- Verified reduction in vulnerabilities and improved system security posture

## Outcome
The implementation and validation of layered security controls reduced overall system risk and improved defensive capabilities against potential threats.

- Enhance continuous monitoring and alerting
- Perform regular vulnerability reassessments
